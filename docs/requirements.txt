# タスク時間割合管理アプリ 要件定義
作成日: 2026-05-24

## 概要
自分が持っている案件に対して、実際の仕事時間の記録をもとに
各案件に何割ずつ時間を使ったかを算出・可視化するツール。

---

## 技術仕様

### 動作環境
- ローカルで動作するブラウザアプリ
- HTML + JavaScript のみで構成（サーバー不要）
- ブラウザ: Chrome 系（File System Access API 対応必須）

### パッケージ管理
- npm パッケージを使用する場合は pnpm を使うこと
- pnpm のネイティブ設定 `minimumReleaseAge` を 7日 に設定すること
  （参照: https://pnpm.io/settings#minimumreleaseage）

### データ保存
- File System Access API を使用してローカル JSON ファイルに直接読み書き
- 起動時に自動でファイル選択ダイアログを開く（2ファイルを別々に選択）
- 選択対象は既存ファイルのみ（存在しないファイルは選択不可）
- ファイルは 2 つ:
  1. presets.json  : 案件プリセット情報を保存
  2. records.json  : 時間記録データを保存

---

## 画面構成（3画面、タブ切り替え）

### 画面1: 案件プリセット管理
- 登録済みプリセットの一覧表示
- 各プリセットに対してできる操作:
  - 追加（新規プリセット登録）
  - 名前の変更（リネーム）
  - 無効化（時間登録の入力候補から外れる。過去記録は残る）
  - 有効化（無効化の逆）
- 無効化された案件は視覚的に区別して表示（一覧には残す）
- 完全削除機能は不要
- 色設定機能は不要

### 画面2: 時間入力（週カレンダー）
- Google カレンダー風の週表示（月〜日の7列）
- 時間軸は縦方向、表示範囲は 7:00〜22:00、15分グリッド
- 日をまたぐ時間入力は不可（同日内のみ）
- ドラッグ操作で時間範囲を指定 → ポップアップで案件を選択して保存
  - ポップアップで選択できる案件は「有効」なもののみ
- 登録済みのブロック（エントリ）をクリック → ポップアップが開き以下が可能:
  - 時間範囲の編集
  - 案件の変更
  - 記録の削除
- 時間の重複（同一時間帯に複数エントリ）は警告して禁止
- 週の前後への移動が可能

### 画面3: 月別割合表示
- 年月の切り替えが可能（前月・翌月ボタン）
- 選択した月の記録を集計
- 無効化された案件の記録も集計・グラフに含める
- 案件ごとの合計時間と割合（%）を表示
- 円グラフ（Pie Chart）で割合を可視化

## UI 仕様

- 言語: 日本語
- レスポンシブ対応: 不要（PC ブラウザ固定想定）
- 外部ライブラリ: Chart.js（円グラフ描画）

## データ形式（JSON スキーマ）

### presets.json
```json
{
  "presets": [
    {
      "id": "string (UUID)",
      "name": "string",
      "enabled": true
    }
  ]
}
```

### records.json
```json
{
  "records": [
    {
      "id": "string (UUID)",
      "presetId": "string (UUID, presets.json の id と対応)",
      "date": "string (YYYY-MM-DD)",
      "startTime": "string (HH:MM, 15分単位)",
      "endTime": "string (HH:MM, 15分単位)"
    }
  ]
}
```
